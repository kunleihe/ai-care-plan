## 1. 项目背景与目标

### 1.1 背景

CVS Specialty Pharmacy 的药剂师目前需要**人工为每位患者生成 Care Plan**，每份耗时 **20–40 分钟**。Care Plan 是合规与医保报销（Medicare 及制药公司）的强制要求。由于严重人手不足，目前已出现积压。

### 1.2 目标

构建一个 Web 工具，使医疗助理能够：

- 通过结构化表单录入患者与订单信息
- 系统自动调用 LLM 生成标准化 Care Plan
- 将 Care Plan 导出为可下载的文本文件，并支持快速导出供制药公司报告使用

### 1.3 预期收益

| 指标 | 现状 | 目标 |
|------|------|------|
| 单份 Care Plan 耗时 | 20–40 分钟 | < 2 分钟（人工审核时间）|
| 积压清除速度 | 受人力限制 | 批量处理 |
| 合规风险 | 因积压而存在漏报风险 | 流程规范化后显著降低 |

## 2. 用户与使用场景

### 2.1 用户群体

| 用户角色 | 说明 |
|----------|------|
| **医疗助理（Medical Assistant）** | 主要操作者，负责录入患者信息并触发生成 |
| **药剂师（Pharmacist）** | 审核生成结果，打印后交给患者 |

> ⚠️ **患者不直接接触本系统。** Care Plan 由工作人员打印后线下交付给患者。

### 2.2 典型工作流

```
医疗助理录入患者信息
        ↓
系统验证数据 + 重复检测
        ↓
（如有 WARNING）工作人员确认是否继续
（如有 ERROR）必须修正后才能继续
        ↓
调用 LLM 生成 Care Plan
        ↓
药剂师审核 → 打印交给患者
        ↓
导出数据用于制药公司报告
```

## 3. 系统输入规格

以下为 Web 表单的输入字段及验证规则：

| 字段名 | 类型 | 验证规则 |
|--------|------|----------|
| Patient First Name | `string` | 必填，非空 |
| Patient Last Name | `string` | 必填，非空 |
| Patient MRN | `string` | 必填，唯一 6 位数字 |
| Patient Date of Birth | `date` | 必填，合法日期，不得为未来日期 |
| Referring Provider | `string` | 必填，非空 |
| Referring Provider NPI | `string` | 必填，恰好 10 位数字（符合 NPI 格式） |
| Patient Primary Diagnosis | `string` | 必填，合法 ICD-10 编码格式 |
| Medication Name | `string` | 必填，非空 |
| Additional Diagnoses | `list<string>` | 可选，每项须为合法 ICD-10 编码 |
| Medication History | `list<string>` | 可选，每项为字符串 |
| Patient Records | `string` 或 `PDF` | 可选，若上传 PDF 则须为有效 PDF 文件 |

## 4. 输出规格

### 4.1 Care Plan 对应关系

- **一个 Care Plan 对应一个订单（一种药物）**
- 同一患者的不同药物需分别生成独立的 Care Plan

### 4.2 Care Plan 必须包含以下四个部分

| 章节 | 说明 |
|------|------|
| **Problem List / Drug Therapy Problems (DTPs)** | 基于患者临床信息，列出药物治疗相关问题 |
| **Goals (SMART)** | 可量化的治疗目标，包括主要目标与安全目标 |
| **Pharmacist Interventions / Plan** | 药剂师干预措施，包含剂量、给药、监测等 |
| **Monitoring Plan & Lab Schedule** | 实验室检查计划与时间节点 |

### 4.3 导出格式

- **Care Plan：** 可下载的 `.txt` 文本文件
- **制药公司报告：** 支持快速导出（格式待定，建议 CSV 或结构化 JSON）

## 5. 功能需求

### 5.1 Web 表单

- [ ] 提供包含所有必填/选填字段的结构化录入表单
- [ ] 所有字段在提交前完成前端验证（格式、必填性）
- [ ] 支持 PDF 文件上传（患者病历）
- [ ] 表单提交后展示验证结果（错误 / 警告）

### 5.2 重复与一致性检测

- [ ] 订单级别重复检测（见第 6 节）
- [ ] 患者级别重复检测（见第 6 节）
- [ ] Provider 一致性校验（NPI 唯一性，见第 7 节）

### 5.3 Care Plan 生成

- [ ] 调用 LLM（Claude API）基于患者记录生成 Care Plan
- [ ] 生成结果展示于页面供审核
- [ ] 支持下载为 `.txt` 文件

### 5.4 报告导出

- [ ] 支持将订单数据快速导出为结构化文件（CSV / JSON）
- [ ] 导出字段覆盖患者基础信息、诊断、药物、Provider 信息

## 6. 重复检测规则

### 6.1 订单级别重复（Order Duplicate）

| 场景 | 处理方式 | 原因 |
|------|----------|------|
| 同一患者 + 同一药物 + **同一天** | ❌ **ERROR — 必须阻止** | 肯定是重复提交 |
| 同一患者 + 同一药物 + **不同天** | ⚠️ **WARNING — 可确认继续** | 可能是续方 |

### 6.2 患者级别重复（Patient Duplicate）

| 场景 | 处理方式 | 原因 |
|------|----------|------|
| MRN 相同 + 名字或 DOB 不同 | ⚠️ **WARNING — 可确认继续** | 可能是录入错误 |
| 名字 + DOB 相同 + MRN 不同 | ⚠️ **WARNING — 可确认继续** | 可能是同一人 |

### 6.3 Provider 一致性

| 场景 | 处理方式 | 原因 |
|------|----------|------|
| NPI 相同 + Provider 名字不同 | ❌ **ERROR — 必须修正** | NPI 是唯一标识符 |

### 6.4 交互逻辑

- **ERROR（❌）**：提交被阻断，用户必须修正数据才能继续
- **WARNING（⚠️）**：弹出确认提示，用户明确确认后方可继续提交

## 7. 数据完整性规则

| 规则 | 说明 |
|------|------|
| MRN 全局唯一 | 同一 MRN 只能关联一位患者 |
| NPI 全局唯一 | 同一 NPI 只能关联一个 Provider 名称（首次录入后不可更改） |
| ICD-10 编码格式 | 所有诊断编码须符合 ICD-10 格式规范 |
| NPI 格式 | 须为恰好 10 位数字 |
| 日期合法性 | DOB 不得为未来日期 |

## 8. 非功能性需求（生产就绪）

| 类别 | 要求 |
|------|------|
| **输入验证** | 每个输入字段均有明确的验证规则，前后端双重校验 |
| **错误处理** | 错误信息安全、清晰、范围可控；不暴露系统内部信息 |
| **数据完整性** | 完整性规则全程强制执行，确保数据一致性 |
| **代码质量** | 代码模块化，结构清晰，易于导航和维护 |
| **自动化测试** | 关键逻辑（验证、重复检测、LLM 调用）覆盖自动化测试 |
| **端到端可运行** | 项目开箱即用，无需额外配置即可端到端运行 |
| **安全性** | 患者数据须符合 HIPAA 相关安全要求（传输加密、访问控制） |

## 10. Care Plan 输出结构

以下为标准输出模板，LLM 须按此结构生成内容：

```
Care Plan
患者姓名：[First Name] [Last Name]
MRN：[MRN]
药物：[Medication Name]
主诊断：[Primary Diagnosis ICD-10]
生成日期：[Date]
Referring Provider：[Provider Name]（NPI：[NPI]）

────────────────────────────────────────

1. Problem List / Drug Therapy Problems (DTPs)
   - [基于患者病历识别的药物治疗问题列表]

2. Goals (SMART)
   - Primary Goal：[主要治疗目标，可量化，含时间节点]
   - Safety Goal：[安全目标]
   - Process Goal：[流程目标]

3. Pharmacist Interventions / Plan
   - Dosing & Administration：[剂量与给药方案]
   - Premedication：[预处理方案（如适用）]
   - Infusion Rates & Titration：[输注速率（如适用）]
   - Hydration & Renal Protection：[水化与肾保护措施]
   - Adverse Event Management：[不良反应处理预案]

4. Monitoring Plan & Lab Schedule
   - 首次给药前：[检查项目列表]
   - 给药期间：[监测频率与项目]
   - 疗程结束后：[复查项目与时间节点]
```
